About my Web-app

This project is a full-stack web application I built to simplify employee attendance tracking. Instead of maintaining records manually, this system provides a structured and interactive way to manage attendance data through a web interface.

I developed this as part of my learning in Computer Science, where I wanted to apply concepts like backend development, database integration, and real-world deployment into a practical project.

The application allows users to add employees, mark their daily attendance, and view records through a dashboard. There is also a weekly view that helps in understanding attendance patterns more clearly. The data is stored in a PostgreSQL database and dynamically rendered using Flask and HTML templates.

From a technical perspective, the backend is built using Python and Flask, with psycopg2 handling database connectivity. The frontend is kept simple using HTML and Bootstrap to ensure clarity and usability. I structured the project in a modular way by separating database logic and service-level operations, which made the code more maintainable and easier to extend.

One of the key parts of this project was deployment. I successfully hosted the application on Render, which helped me understand how real-world applications are made accessible online.

Through this project, I gained hands-on experience in building CRUD-based systems, working with relational databases, structuring backend services, and managing version control using Git and GitHub. I also learned how small details like clean code, comments, and project structure make a big difference in overall quality.

You can try the live application here:
https://attendance-tracker-bk-agencies.onrender.com/

Future improvements I plan to work on include adding user authentication, exporting attendance reports, and improving the overall UI/UX to make it more polished.

This project represents a step forward in my journey of building practical, real-world applications.
